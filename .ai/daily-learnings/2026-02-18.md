### 1. Capacitor Static Export

Next.js 앱을 Capacitor로 iOS 네이티브 앱으로 래핑할 때, `output: "export"` 설정으로 정적 HTML/JS/CSS를 `out/` 디렉토리에 생성한 후, `cap sync`로 iOS 프로젝트의 `public/` 폴더에 복사한다.

```js
// next.config.mjs
const isCapacitor = process.env.NEXT_PUBLIC_BUILD_TARGET === "capacitor";

const nextConfig = {
  output: isCapacitor ? "export" : undefined,
  images: { unoptimized: isCapacitor },
};
```

- `output: "export"` 사용 시 API Routes, Middleware, 서버 사이드 기능 사용 불가
- 이미지 최적화도 비활성화해야 함 (`unoptimized: true`)
- `rewrites()`도 Capacitor에서는 동작하지 않으므로 직접 API URL 사용

### 2. xcodebuild CLI 빌드

Xcode GUI를 열지 않고도 커맨드라인에서 iOS 앱을 빌드할 수 있다. CI/CD 파이프라인이나 자동화에 유용하다.

```bash
# xcworkspace가 아닌 xcodeproj를 사용하는 경우 (SPM 기반)
xcodebuild \
  -project App.xcodeproj \
  -scheme App \
  -destination 'platform=iOS Simulator,id=<SIMULATOR_UUID>' \
  -allowProvisioningUpdates \
  build
```

- `-workspace` vs `-project`: CocoaPods를 쓰면 `.xcworkspace`, SPM이면 `.xcodeproj`
- `-destination`: 시뮬레이터 UUID 또는 이름으로 타겟 지정
- `-allowProvisioningUpdates`: 시뮬레이터에서는 "Sign to Run Locally"로 자동 처리됨

### 3. xcrun simctl

`xcrun simctl`은 iOS 시뮬레이터를 CLI에서 직접 제어하는 도구이다.

```bash
# 사용 가능한 시뮬레이터 목록 확인
xcrun simctl list devices available | grep "iPhone"

# 시뮬레이터 부팅
xcrun simctl boot <SIMULATOR_UUID>

# 앱 설치
xcrun simctl install <SIMULATOR_UUID> /path/to/App.app

# 앱 실행 (번들 ID 필요)
xcrun simctl launch <SIMULATOR_UUID> com.gotcha.it.app
```

- 번들 ID는 `Info.plist`의 `CFBundleIdentifier` 값과 일치해야 함
- `PlistBuddy`로 빌드된 앱에서 번들 ID를 확인할 수 있음:
  ```bash
  /usr/libexec/PlistBuddy -c "Print :CFBundleIdentifier" App.app/Info.plist
  ```

### 4. SPM 기반 Capacitor 프로젝트

Capacitor 6+에서는 CocoaPods 대신 Swift Package Manager(SPM)를 기본으로 사용한다.

- `ios/App/CapApp-SPM/Package.swift`에서 플러그인 의존성 관리
- `ios/App/Pods/` 대신 Xcode가 SPM 패키지를 자동 resolve
- `.xcworkspace` 없이 `.xcodeproj`만으로 빌드 가능
- `cap sync` 실행 시 `Package.swift`가 자동 업데이트됨

### 5. Client/Page 분리 패턴

Next.js의 `output: "export"` 모드에서 동적 라우트(`[id]`)를 사용하려면 `generateStaticParams()`가 필요하다. 서버 컴포넌트인 `page.tsx`에서 `generateStaticParams()`를 제공하고, 실제 UI 로직은 `"use client"` 컴포넌트로 분리한다.

```tsx
// page.tsx (서버 컴포넌트)
import ShopDetailClient from "./ShopDetailClient";

export function generateStaticParams() {
  return [{ id: "0" }]; // 정적 내보내기를 위한 더미 파라미터
}

export default function ShopDetailPage({ params }: { params: { id: string } }) {
  return <ShopDetailClient />;
}
```

```tsx
// ShopDetailClient.tsx (클라이언트 컴포넌트)
"use client";
export default function ShopDetailClient() {
  const params = useParams();
  // 실제 UI 로직...
}
```

- `generateStaticParams()`는 빌드 시 정적으로 생성할 경로를 지정
- 더미 값 `[{ id: "0" }]`만 넣어도 Capacitor에서 클라이언트 사이드 라우팅이 처리
- 서버 사이드 `redirect()`도 클라이언트 `useRouter().replace()`로 전환 필요
