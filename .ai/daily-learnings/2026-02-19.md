### 1. Apple APNs p8 키

iOS 푸시 알림을 보내려면 백엔드에서 Apple Push Notification Service(APNs)와 통신해야 한다. p8 키는 JWT 기반 인증 방식으로, 만료 없이 재사용 가능하다.

**백엔드에 전달할 정보:**

- p8 키 파일 (`.p8`)
- Key ID (예: `RF5MXDT26R`)
- Team ID (예: `6KCGZ5594F`)
- Bundle ID (예: `com.it.gotcha.app`)

**p8 vs p12 차이:**

- p8: JWT 기반, 만료 없음, 하나의 키로 여러 앱 지원
- p12: 인증서 기반, 1년 만료, 앱별 발급 필요

### 2. Capacitor iosScheme 제한

Capacitor의 `iosScheme` 설정은 WKWebView의 커스텀 URL 스킴을 지정한다. 하지만 **WKWebView가 이미 처리하는 스킴은 사용 불가**하다.

```swift
// CAPInstanceDescriptor.swift (Capacitor 내부 코드)
if let scheme = urlScheme, WKWebView.handlesURLScheme(scheme) == false {
    schemeValid = true
}
// https, http, ftp 등은 handlesURLScheme()이 true를 반환 → 사용 불가
// 기본값 "capacitor"로 리셋됨
```

- `iosScheme: "https"` → **작동하지 않음** (기본값 `capacitor://`로 리셋)
- `iosScheme: "myapp"` → 작동함 (커스텀 스킴)
- 앱 삭제 후 재설치해야 스킴 변경이 적용됨

### 3. Capacitor server.url

로컬 정적 파일 대신 원격 웹앱 URL을 WebView에 로드할 수 있다.

```ts
// capacitor.config.ts
const config: CapacitorConfig = {
  server: {
    url: "https://dev.gotcha.it.com", // 원격 웹앱 로드
  },
};
```

**장점:**

- origin이 `https://dev.gotcha.it.com` → 카카오 지도, CORS, OAuth 모두 정상
- Capacitor 네이티브 플러그인(푸시, 위치 등)은 그대로 사용 가능

**단점:**

- 인터넷 필수 (오프라인 불가)
- 배포된 웹앱 코드 사용 (로컬 변경사항 미반영)
- SplashScreen `launchAutoHide: false` 시 스플래시에서 멈출 수 있음

### 4. WKWebView 이중 팝업 문제

Capacitor iOS 앱에서 `navigator.geolocation`이나 `Notification.requestPermission()`을 사용하면 WKWebView 팝업 + iOS 네이티브 팝업이 동시에 뜬다.

**해결:** Capacitor 네이티브 플러그인으로 대체

```ts
// 웹: 이중 팝업 발생
navigator.geolocation.getCurrentPosition(success, error);

// 네이티브: 단일 iOS 팝업만 표시
const { Geolocation } = await import("@capacitor/geolocation");
const position = await Geolocation.getCurrentPosition({ enableHighAccuracy: true });
```

- `@capacitor/geolocation` → 위치 권한 (단일 팝업)
- `@capacitor/push-notifications` → 알림 권한 (단일 팝업)

### 5. 카카오 지도 SDK 도메인 검증

카카오 지도 JavaScript SDK는 `kakao.maps.load()` 호출 시 도메인을 검증한다. `capacitor://localhost`는 유효한 도메인으로 인식하지 않는다.

**증상:**

- 스크립트 다운로드: 성공 (`window.kakao.maps` 존재)
- `kakao.maps.load()` 콜백: **호출되지 않음** (도메인 검증 실패)
- 카카오 개발자 콘솔에 `capacitor://localhost` 등록해도 무효

**해결:** `server.url`로 등록된 도메인(`https://dev.gotcha.it.com`)에서 로드

### 6. Capacitor 네이티브 플러그인 분기

`isNativeApp()`으로 런타임에 웹/네이티브를 분기하여 적절한 API를 사용한다.

```ts
import { isNativeApp } from "@/utils/platform";

if (isNativeApp()) {
  // Capacitor 플러그인 (동적 import로 웹에서 번들 미포함)
  const { Geolocation } = await import("@capacitor/geolocation");
  const position = await Geolocation.getCurrentPosition();
} else {
  // 웹 표준 API
  navigator.geolocation.getCurrentPosition(callback);
}
```

- 동적 `import()`로 Capacitor 플러그인을 로드하면 웹 빌드에 포함되지 않음
- `isNativeApp()`은 `window.Capacitor?.isNativePlatform()`을 내부적으로 호출

### 7. NEXT_PUBLIC_BUILD_TARGET 환경변수

`.env.local` 파일을 수정하지 않고 빌드 명령어에서 환경변수를 직접 전달한다.

```bash
# .env.local 수정 없이 Capacitor 빌드
NEXT_PUBLIC_BUILD_TARGET=capacitor npx next build

# package.json에 스크립트로 등록
"build:capacitor": "NEXT_PUBLIC_BUILD_TARGET=capacitor next build"
"build:ios": "npm run build:capacitor && npx cap sync ios && npx cap open ios"
```

- `npm run dev` 시에는 웹 모드로 동작 (영향 없음)
- Capacitor 빌드 시에만 `output: "export"`, `images.unoptimized: true` 적용
